# Testing Rules

1. Test immediately after building each unit.
2. Add tests before or alongside implementation when behavior is clear.
3. Backend tests must use Vitest, Supertest, and MongoDB Memory Server.
4. Frontend components must be browser-verified or tested for all important states.
5. Do not claim a feature is complete unless verification ran or the blocker is stated.
6. Frontend verification must include 320px and 375px mobile widths.
