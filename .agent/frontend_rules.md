# Frontend Rules

1. Build usable app screens, not placeholder marketing pages.
2. Each component must handle loading, empty, error, disabled, active, overflow, mobile, and desktop states.
3. Use existing design tokens and CSS variables.
4. Prefer lucide icons.
5. Do not nest cards.
6. Do not use text where an obvious icon button is better.
7. Avoid one-color themes and decorative filler.
8. Verify visual output in browser after meaningful UI work.
9. Mobile responsiveness is mandatory.
10. Verify every page at 320px and 375px widths.
11. No horizontal scrolling, clipped controls, overlapping text, or unreachable map/form actions on small screens.
