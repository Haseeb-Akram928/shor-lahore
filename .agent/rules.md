# Agent Rules

1. Read `AGENTS.md` first.
2. Read the project context files before coding.
3. Follow the current phase plan in `docs/plans/`.
4. Build one small unit at a time.
5. Test immediately after building.
6. Cover success and failure states.
7. Do not skip edge cases.
8. Keep TypeScript strict.
9. Do not add unnecessary libraries.
10. Keep final responses brief.
